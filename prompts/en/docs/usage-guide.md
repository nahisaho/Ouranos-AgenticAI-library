## 基本的な使い方

### ステップ1: 目的の明確化

まず、何を達成したいのかを明確にします。

**質問例:**
- どのような成果物が欲しいですか？
- どのような課題を解決したいですか？
- 誰のために、何のために使用しますか？

### ステップ2: カテゴリの選択

目的に応じて適切なカテゴリを選択します。

| 目的 | 推奨カテゴリ |
|------|-------------|
| ビジネス戦略を立てたい | `business-management/` |
| ソフトウェアを設計したい | `design-development/` |
| チームを構築したい | `hr-organization/` |
| カリキュラムを作成したい | `education/` |
| 汎用的なプロンプトが欲しい | `general/` |

### ステップ3: プロンプトファイルの参照

選択したカテゴリ内のマークダウンファイルを開きます。

```bash
# 例: メタプロンプト生成エージェントを使用
cat prompts/general/meta-prompt-generator.md
```

### ステップ4: AIへの適用

プロンプトの内容をAIチャット（ChatGPT、Claude、Geminiなど）にコピー&ペーストします。

**推奨:**
- YAMLフロントマター以降の内容をコピー
- または全体をコピーしてもOK（AIは自動的にYAML部分を認識）

### ステップ5: 対話の実行

AIとの対話を通じて、目的に応じたメタプロンプトや成果物を生成します。

## プロンプトの選択方法

### manifest.ymlを活用

`manifest.yml`を参照して、全プロンプトの概要を把握できます。

```bash
# manifest.ymlを表示
cat manifest.yml
```

**manifest.ymlの読み方:**
- `id`: プロンプトの識別子
- `category`: カテゴリ分類
- `frameworks`: 使用しているフレームワーク
- `tags`: 検索用タグ

### タグで検索

特定のタグでプロンプトを検索できます。

```bash
# "strategy"タグを含むプロンプトを検索
grep -r "strategy" prompts/
```

### フレームワークで選択

使いたいフレームワークが決まっている場合:

```bash
# SWOT分析を使用するプロンプトを検索
grep -r "SWOT" prompts/
```

## プロンプトの活用方法

### パターン1: そのまま使用

プロンプトをそのままコピー&ペーストして使用します。

**メリット:**
- すぐに使える
- 一貫した品質

**適用場面:**
- 初めて使う場合
- 標準的なアプローチで十分な場合

### パターン2: 部分的に使用

必要なセクションのみを抽出して使用します。

**例:**
- 「対話フロー」のみを参照して、自分で質問を組み立てる
- 「フレームワーク」のみを参照して、別のプロンプトに統合する

### パターン3: カスタマイズして使用

自分の目的に合わせてプロンプトを修正します。

詳細は [カスタマイズ方法](#カスタマイズ方法) を参照してください。

## カスタマイズ方法

### 基本的なカスタマイズ

1. **役割の変更**
   - AIの専門性や立場を変更
   - 例: "経営コンサルタント" → "スタートアップアドバイザー"

2. **対話ステージの追加・削除**
   - 目的に応じてステージを調整
   - 例: 詳細な分析が必要な場合はステージを追加

3. **フレームワークの変更**
   - 使用するフレームワークを入れ替え
   - 例: SWOT分析 → VRIO分析

4. **出力形式の変更**
   - 期待する成果物の形式を変更
   - 例: レポート形式 → プレゼンテーション形式

### 高度なカスタマイズ

1. **複数プロンプトの統合**
   - 異なるカテゴリのプロンプトを組み合わせる
   - 例: ビジネス戦略 + 教育設計

2. **業界特化型への変換**
   - 特定の業界向けにカスタマイズ
   - 業界用語や慣習を追加

3. **多段階プロセスの設計**
   - 複数のプロンプトを連鎖させる
   - 各ステージで異なるプロンプトを使用

### カスタマイズ例

**元のプロンプト（一部）:**

```markdown
## 役割
あなたは経験豊富なビジネス戦略コンサルタントです。
```

**カスタマイズ後:**

```markdown
## 役割
あなたは医療業界に特化した経験豊富なビジネス戦略コンサルタントです。
医療機関の経営改善と患者満足度向上の両立を支援することが使命です。
```

## ベストプラクティス

### 対話を最大限に活用する

1. **具体的に答える**
   - AIの質問には具体的な情報を提供
   - 曖昧な回答は質の低い結果につながる

2. **反復的に改善する**
   - 一度の対話で完璧を求めない
   - フィードバックを与えて改善を重ねる

3. **サンプルを提供する**
   - 期待する成果物の例を示す
   - 既存の資料を参考として提供

### 効果的なプロンプト利用

1. **コンテキストを明確にする**
   - 背景情報を十分に提供
   - 制約条件を事前に伝える

2. **段階的に進める**
   - 一度に全てを求めない
   - ステージごとに確認しながら進める

3. **記録を残す**
   - 対話の内容を保存
   - 後で参照・再利用できるようにする

### 避けるべきこと

1. **❌ 曖昧な指示**
   - 「良い感じに」「適当に」などの曖昧な表現は避ける

2. **❌ 情報不足**
   - 必要最低限の情報しか提供しない

3. **❌ 一方的な期待**
   - AIに全てを任せず、対話的に協力する

4. **❌ コンテキストの欠如**
   - 背景や目的を説明せずにいきなり詳細を聞く

## トラブルシューティング

### Q: 期待した結果が得られない

**A: 以下を確認してください**
1. 目的が明確に伝わっているか
2. 十分なコンテキスト情報を提供したか
3. 適切なプロンプトを選択したか
4. 対話の各ステージを丁寧に進めたか

### Q: プロンプトが複雑すぎる

**A: シンプルなものから始めてください**
1. `general/meta-prompt-generator.md`から開始
2. 必要に応じて専門的なプロンプトに移行
3. 部分的に使用することも検討

### Q: カスタマイズ方法がわからない

**A: 段階的にアプローチしてください**
1. まずはそのまま使ってみる
2. 小さな変更から始める（役割の変更など）
3. 徐々に大きなカスタマイズに挑戦
