# Examples for Agile Coach

This file provides concrete usage examples and scenarios demonstrating this prompt in action.

---

### Scenario: Software Development Team Agile Adoption

**Stage 1: Assessment**
- Team: 8-person development team, waterfall background
- Challenge: Long release cycles, quality issues, low morale
- Context: Management supports change, no dependencies

**Stage 2: Framework Selection**
- Chose: Scrum (provides structure for beginners)
- Rationale: Team needs discipline, stakeholders want predictability
- Tailored: 2-week sprints, simplified ceremonies initially

**Stage 3: Practices Setup**
- Sprint Planning: Wednesdays 9am-11am
- Daily Scrum: Daily 9:15am, 15 minutes
- Sprint Review: Tuesday 2pm, demo to stakeholders
- Sprint Retrospective: Tuesday 3pm, team only
- Definition of Done: Code reviewed, tested, documented, deployed to staging

**Stage 4: Metrics**
- Velocity: Started at 20 points, stabilized at 35 points by sprint 5
- Quality: Defects reduced from 15/sprint to 3/sprint
- Morale: Happiness score improved from 2.5/5 to 4.2/5

**Stage 5: Transformation**
- Month 1-2: Learning, velocity unstable
- Month 3-4: Practices becoming routine, velocity stabilizing
- Month 5-6: Continuous improvement, adding TDD
- Results: 50% faster releases, 80% fewer defects, team satisfaction high
