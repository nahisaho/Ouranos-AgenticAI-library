This file provides concrete usage examples and scenarios demonstrating this prompt in action.

### Scenario: Mobile Banking App Redesign

**Stage 1: Design Challenge**
- Product: Mobile banking app for millennials
- Problem: Low engagement, confusing navigation, outdated UI
- Goal: Increase daily active users by 40%

**Stage 2: User Research**
- Methods: 12 user interviews, analytics analysis, competitor analysis
- Personas: "Busy Professional Mia" (primary), "Budget-Conscious Ben" (secondary)
- Key insight: Users want quick access to balance, not complex features

**Stage 3: IA and Interaction**
- Simplified navigation: Home, Transfer, Insights, Profile
- Primary user flow: Check balance in 1 tap
- Card-based interface for scannable content

**Stage 4: Visual Design**
- Modern, clean aesthetic with financial brand trust
- Green accent color for actions, red for warnings
- Large, readable typography (16px minimum)
- Component library with 30+ reusable components

**Stage 5: Usability Testing**
- 3 rounds with 6 participants each
- Task completion rate improved from 60% to 95%
- Time to check balance reduced from 8 seconds to 2 seconds
- SUS score increased from 52 to 81

## Dialogue Quality Tips

### Creating Natural Flow

Build on user responses naturally, showing domain expertise and genuine engagement.

### Demonstrating Expertise

Use domain-specific knowledge to provide valuable insights and guidance.

### Adaptive Responses

Adjust depth and complexity based on user's expertise level and responses.
