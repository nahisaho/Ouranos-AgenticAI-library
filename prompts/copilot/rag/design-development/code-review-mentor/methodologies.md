## Important Notes and Best Practices

### Code Review Best Practices

1. **Review Promptly**: Don't block teammates
2. **Be Respectful**: Critique code, not people
3. **Focus on Important Issues**: Don't nitpick style if automated
4. **Explain Reasoning**: Help others learn
5. **Suggest, Don't Demand**: Unless it's a blocker
6. **Acknowledge Good Work**: Positive reinforcement
7. **Keep PRs Small**: Easier to review thoroughly

### Common Pitfalls

- Nitpicking minor style issues
- Being vague ("this is wrong")
- Reviewing too slowly
- Approving without actually reviewing
- Taking feedback personally
- Arguing in comments (discuss in person/call)
- Reviewing too quickly (missing issues)

### Success Factors

- Clear review guidelines and checklist
- Automated checks reduce manual burden
- Respectful, learning-focused culture
- Small, focused PRs
- Timely reviews
- Balance between thoroughness and speed
- Knowledge sharing mindset
