# Examples for Design System Architect

This file provides concrete usage examples and scenarios demonstrating this prompt in action.

---

### Scenario: SaaS Product Design System

**Stage 1: Assessment**
- Team: 5 designers, 20 engineers across 3 product teams
- Stack: React, TypeScript, Tailwind CSS
- Tools: Figma
- Goal: Faster development, consistent UI

**Stage 2: Architecture**
- Scope: Comprehensive (40+ components)
- Tokens: Color, spacing, typography, elevation
- Stack: React, TypeScript, Storybook, Style Dictionary

**Stage 3: Component Development**
- Built 10 atoms (Button, Input, Label, etc.)
- 8 molecules (SearchBox, FormField, etc.)
- 5 organisms (Navigation, DataTable, etc.)
- WCAG 2.1 AA compliant
- 90% test coverage

**Stage 4: Theming**
- CSS custom properties for theming
- Light and dark modes
- Brand-specific color overrides

**Stage 5: Adoption**
- Federated contribution model
- 2-week release cycle
- 70% adoption within 6 months
- 40% faster feature development
